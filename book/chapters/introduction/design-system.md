## Links

## Code Blocks

A fenced block should _almost always_ have a language identifier for syntax highlighting.

For ColdFusion script _with_ the `<cfscript>` wrapper, use `cfm`:

```cfm
<cfscript>
    public string function greet( required string name ) {
        return "Hello, #name#!";
    }
</cfscript>
```

For ColdFusion script _without_ the `<cfscript>` wrapper, use `cfs`:

```cfs
public string function greet( required string name ) {
    return "Hello, #name#!";
}
```

For ColdFusion components, use `cfc`:

```cfc
component {
    this.name = "MyApplication";
}
```

For mixed CFML with embedded HTML/CSS/JavaScript, use `cfm`:

```cfm
<cfoutput>
    <p>
        This is #encodeForHtml( description )#!
    </p>
    <script type="text/javascript">
        console.log( "Page loaded!" );
    </script>
</cfoutput>
```

To highlight specific lines, use the `{highlight=""}` attribute. It uses a comma-delimited list that supports single lines and line-ranges:

```cfc {highlight="2,5-6"}
component {
    property name="dataSource";

    function query() {
        var sql = "SELECT * FROM user";
        var result = queryExecute( sql );
        return result;
    }
}
```

To label the source file for your code block, use the `{file=""}` attribute. Note that the `file` and `highlight` attributes can be combined:

```cfc {file="models/UserService.cfc" highlight="2"}
component {
    property name="dataSource";

    function query() {
        var sql = "SELECT * FROM user";
        var result = queryExecute( sql );
        return result;
    }
}
```

## Callouts

Callouts are editorial notes that render as `<aside>` elements. We support the same callouts that are defined in the GitHub flavor of Markdown.

For an informational note, use the `[!NOTE]` marker:

> [!NOTE]
> Useful information that users should know, even when skimming content.

For a helpful note, use the `[!TIP]` marker:

> [!TIP]
> Helpful advice for doing things better or more easily.

For an important note, use the `[!IMPORTANT]` marker:

> [!IMPORTANT]
> Key information users need to know to achieve their goal.

For a warning note, use the `[!WARNING]` marker:

> [!WARNING]
> Urgent info that needs immediate user attention to avoid problems.

For a cautionary note, use the `[!CAUTION]` marker:

> [!CAUTION]
> Advises about risks or negative outcomes of certain actions.
